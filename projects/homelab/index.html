HomeLab

A production-grade home server built on an M4 Mac Mini running a suite of self-hosted services behind a private VPN mesh. Designed around privacy, data ownership, and developer productivity — with zero reliance on third-party cloud subscriptions for core workflows.

Architecture Overview

Most services are containerized with Docker and managed through Portainer. Secure remote access is handled by a Tailscale mesh VPN, with Caddy acting as a reverse proxy for HTTPS routing to each service. A unified Homepage dashboard provides a single pane of glass across the entire stack. AI inference workloads run via Ollama and llama.cpp, surfaced through Open WebUI as a unified chat and tooling interface.

Networking

Tailscale

A WireGuard-based mesh VPN that connects all devices — laptop, phone, and server — into a single private network without opening any ports on the host router. Each service is accessible from anywhere as if on a local network.

  • Tailscale HTTPS Serve — provisions valid TLS certificates for Tailscale nodes, enabling HTTPS for all internal services without a public domain or manual certificate management.

Caddy

A modern reverse proxy used to route HTTPS traffic to each containerized service by hostname. Caddy handles automatic certificate renewal and TLS termination, keeping all service endpoints secured with minimal configuration.

ProtonVPN

Outbound VPN for general internet traffic from the host. Ensures the server's external IP is masked for privacy on egress connections.

AI Stack

The homelab runs a fully local AI stack — no external API keys required for core inference. Models are served through two backends depending on use case, both exposed to Open WebUI as a unified interface.

🤖 Open WebUI

Self-hosted AI chat interface deployed via Docker. Open WebUI acts as the primary frontend for all local LLM interaction, connecting to multiple inference backends simultaneously:

  • Ollama API — serves quantized open-weight models (Llama, Mistral, Qwen, etc.) with native Apple Silicon acceleration via Metal. Used for general chat, coding assistants, and tool-calling workflows.
  • llama.cpp API — serves GGUF-format models with fine-grained control over context length, quantization level, and memory allocation. Used when direct model configuration is required beyond what Ollama exposes.
  • RAG (Retrieval-Augmented Generation) — Open WebUI's built-in RAG pipeline is connected to a local document store ingested from the Obsidian vault. Markdown notes, references, and processed documents are chunked, embedded, and stored in a vector database, enabling context-aware queries against personal knowledge directly from the chat interface.

🔍 Vane (formerly Perplexica)

Self-hosted AI-powered search engine deployed via Docker. Vane is an open-source Perplexity alternative that combines web search with LLM synthesis — retrieving live search results and passing them through a local Ollama model to generate cited, summarized answers. Keeps AI-assisted web research entirely local and private.

💻 OpenCode

Terminal-based AI coding assistant integrated with the local Ollama API. OpenCode provides an interactive CLI coding agent — capable of reading, editing, and reasoning about codebases — using locally served models rather than cloud APIs. Used as a self-hosted alternative to cloud coding assistants for private repository work.

🗣 Pi

Conversational AI assistant integrated with Ollama. Pi provides a persistent, persona-driven chat interface backed by a locally running model, used for open-ended reasoning, research assistance, and ideation workflows within the homelab.

🎨 ComfyUI

Node-based Stable Diffusion interface for local AI image generation. Runs bare metal on macOS with Metal GPU acceleration via PyTorch's MPS backend, giving it direct access to the M4's GPU without containerization overhead. ComfyUI's graph-based workflow system enables reproducible, version-controlled generation pipelines stored as JSON.

Knowledge Management

🧠 Obsidian

Primary second brain and knowledge management system — not a hosted service, but a central node in the homelab's information architecture. The Obsidian vault serves multiple roles:

  • Personal knowledge base — notes, research, references, and project documentation are stored as plain Markdown files, version-controlled via Gitea through the Obsidian Git plugin.
  • AI retrieval corpus — vault content is ingested into Open WebUI's RAG pipeline, making personal notes and accumulated knowledge queryable through local LLMs. This creates a closed loop where research done in Obsidian becomes context for AI-assisted workflows.
  • Skills and workflow storage — structured notes define repeatable processes, prompt templates, and domain knowledge used to condition model behavior in AI pipelines.
  • Library and reading tracker — PDFs and academic papers are stored in an Attachments/ directory with corresponding Markdown pages tracking reading progress, highlights, quotes, and author metadata under a unified Library/ structure.

Vault backups are automated via Obsidian Git, committing and pushing to a private Gitea repository on a configurable schedule — providing full version history for all notes.

Services

🖼 Immich

Self-hosted photo and video backup — a drop-in replacement for Google Photos. Deployed via Docker Compose, Immich provides automatic mobile backups, facial recognition, album management, and a web UI. All media stays on-premises with no third-party access.

🗂 Gitea

Self-hosted Git service — a lightweight alternative to GitHub. Used as the primary remote for all personal and homelab repositories, with full support for webhooks, issues, and access tokens.

  • Integrated with Obsidian Git for automated Obsidian vault backups and version control, pushing vault snapshots to a private Gitea repo on a configurable schedule.

🔖 Linkwarden

Self-hosted bookmark and link archive manager. Captures web pages with full-page screenshots and readable content, organized by collection and tags. Replaces browser bookmarks and cloud bookmark services, keeping a permanent, searchable archive of saved links.

📚 Kavita

Self-hosted digital library server for ebooks, comics, and manga. Provides a clean reader UI, reading progress tracking, and OPDS support. Serves the local ebook collection over the Tailscale network to any device.

🐳 Portainer

Docker management UI for the entire container stack. Provides a visual interface for managing containers, images, volumes, networks, and Docker Compose stacks — used for deployment, monitoring, and quick log inspection.

🏠 Homepage

Unified service dashboard (gethomepage.dev) configured as the homelab's entry point. Aggregates live status, stats, and quick links for every running service. Widgets pull live data directly from service APIs — container counts from Portainer, photo counts from Immich, repo counts from Gitea, etc.

Key Design Decisions

Apple Silicon — The M4 Mac Mini's unified memory architecture and power efficiency (~20W idle) make it well-suited as a 24/7 homelab host. Metal GPU acceleration benefits AI workloads in ComfyUI without discrete GPU overhead.

Tailscale over self-managed WireGuard — Eliminates the operational complexity of key distribution and NAT traversal while retaining WireGuard's performance and security model. No ports exposed to the public internet.

Caddy over NGINX — Automatic HTTPS, minimal configuration surface, and native support for Tailscale HTTPS certificates reduces the maintenance burden for TLS across services.

Gitea over GitHub for private repos — Full control over repository data and access, with no dependency on an external service for personal and homelab infrastructure code.
