# HealthPath

A responsive single-page application designed for patients managing chronic health conditions. HealthPath brings together meal planning, health tracking, and provider discovery into a single secure platform — with a clean sage clinical design system that works across light and dark modes.

---

## Features

### Meal Planner
The core feature of the platform. Two planning modes cover different user needs:

- **Quick Mode** — Select a health goal (Heart Health, Kidney Health, Diabetic Friendly, Low Sodium, General Wellness) and generate a full 7-day plan instantly. Each day covers breakfast, lunch, dinner, and a snack.
- **Advanced Mode** — Filter by health condition (CKD stages 1–5, hypertension, type 2 diabetes, heart disease), dietary preference (vegetarian, vegan, gluten-free, dairy-free), nutrients to limit (sodium, potassium, phosphorus, saturated fat, sugar), and allergens. Plans are regenerated on demand.

The meal dataset covers 88+ meals across all categories, each tagged with dietary flags, allergen data, and nutrient avoid tags to power the Advanced filter logic.

### Dashboard
Personalized landing page for signed-in users featuring health stats, a recent activity feed, quick navigation to the meal planner, and contextual health tips.

### Authentication
Full sign up / log in / log out flow with role selection (patient, healthcare provider, caregiver). Sessions persist across page refreshes. The auth layer is localStorage-based — appropriate for a frontend demo without a backend.

## Design System

The palette is a **Sage Clinical** theme built on CSS custom properties, giving both light and dark modes their own distinct character while staying in the same hue family.

- **Light — Sage Mist:** sage-tinted sidebar (`#E8EDE6`), warm white panels, muted sage borders
- **Dark — Deep Forest:** deep forest background (`#141C18`), rich forest panels, sage-cream text
- **Accent:** muted sage green (`#5A8A6A`) — the single cool-adjacent element against the warm neutral surfaces

Theming is implemented with a `darkMode: 'class'` Tailwind config backed by CSS custom properties in RGB channel format, which enables Tailwind's opacity modifiers (`bg-sidebar/10`) to work correctly across both modes. A flash-prevention inline script in `index.html` applies the saved theme class before paint.

## Architecture

```
src/
├── context/
│   ├── AuthContext.jsx     # localStorage auth — login, signup, logout
│   └── ThemeContext.jsx    # light/dark toggle, persisted to localStorage
├── components/
│   └── layout/
│       ├── AppLayout.jsx   # authenticated shell — owns mobile drawer state
│       ├── Sidebar.jsx     # desktop fixed sidebar + mobile right-slide drawer
│       └── TopBar.jsx      # logo / page title / theme toggle / hamburger
├── pages/
│   ├── Landing.jsx         # marketing page, auth-aware nav
│   ├── Login.jsx
│   ├── Signup.jsx
│   ├── Dashboard.jsx
│   └── MealPlanner.jsx
└── data/
    └── mealData.js         # 88+ meals, getQuickPlan(), getAdvancedPlan()
```

The authenticated layout (`AppLayout`) centralizes mobile navigation state and passes it down to `Sidebar` (drawer) and `TopBar` (hamburger). On mobile, the drawer slides in from the right; on desktop, the sidebar is a fixed 240px column.

## Deployment

GitHub Actions builds the Vite app on every push to `main` and deploys to a `gh-pages` branch. A `404.html` copy of `index.html` is included in the build output so that React Router handles direct navigation to any route on GitHub Pages.

```
main push
  └── npm ci → npm run build
       └── cp dist/index.html dist/404.html
            └── peaceiris/actions-gh-pages → gh-pages branch
```
